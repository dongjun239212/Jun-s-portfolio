# 方案二详细步骤：阿里云 OSS + CDN 部署静态站

按顺序做完以下步骤即可让网站在国内通过 OSS + CDN 访问。

> **重要**：使用 **OSS Bucket 自有域名**（如 `xxx.oss-cn-beijing.aliyuncs.com`）访问时，阿里云会强制把 HTML 当作附件下载或导致白屏，这是平台限制，无法通过上传时设置响应头修复。  
> **分享网站请用**：GitHub Pages 或 Vercel 链接（见项目根目录 **`访问说明.md`**）。  
> 要用 OSS 做国内加速且正常打开网页，必须完成 **第 6 步：绑定已备案的自定义域名**，通过自定义域名访问。

---

## 第 0 步：本地构建（必须先做）

在项目目录执行：

```bash
cd "/Users/bytedance/Desktop/My personal website/my-portfolio"
npm run build
```

构建完成后，打开本地 **`out`** 文件夹，确认里面有：
- `index.html`（在 out 根目录）
- `_next/` 文件夹
- `detail/` 文件夹
- 以及其它静态资源

后面上传的是 **`out` 里面的所有内容**，不是 `out` 这个文件夹本身。

---

## 第 1 步：登录阿里云并进入 OSS

1. 打开 **https://www.aliyun.com**，登录账号（没有就注册）。
2. 右上角点击 **控制台** 进入阿里云控制台。
3. 顶部搜索框输入 **对象存储 OSS**，点击进入 **对象存储 OSS** 产品页。
4. 左侧菜单点击 **Bucket 列表**（或「存储空间列表」）。

---

## 第 2 步：创建 Bucket

1. 在 Bucket 列表页点击 **创建 Bucket**。
2. 按下面填写（没提到的保持默认即可）：

   | 配置项 | 填写说明 |
   |--------|----------|
   | **Bucket 名称** | 全局唯一，例如：`my-portfolio` 或 `yourname-portfolio`（若提示被占用换一个）。 |
   | **地域** | 选 **华东1（杭州）** 或 **华北2（北京）** 等离你/用户近的。 |
   | **存储类型** | **标准存储**。 |
   | **存储冗余** | 本地冗余即可。 |
   | **读写权限** | 选 **公共读**（否则网站无法被访问）。 |
   | **服务端加密** | 不开启即可。 |

3. 点击 **确定** 完成创建。
4. 在 Bucket 列表中点击刚创建的 Bucket 名称，进入该 Bucket 管理页。

---

## 第 3 步：开启静态页面（默认首页）

1. 在 Bucket 管理页，左侧菜单找到 **基础设置**（或「数据管理」→「基础设置」）。
2. 找到 **静态页面** 或 **静态网站托管** 相关设置并 **开启**。
3. 填写：
   - **默认首页**：`index.html`
   - **默认 404 页**：`index.html`（这样访问 `/detail` 等路径会由前端路由处理）
4. **保存**。

---

## 第 4 步：上传网站文件

目标：把本地 **`out` 目录里的所有文件和文件夹** 上传到 Bucket 的**根目录**。

### 方式 A：控制台网页上传（适合首次、文件不多）

1. 在 Bucket 管理页，左侧点击 **文件管理**（或「对象」/「Object」）。
2. 确认当前路径是 **根目录**（路径显示为 `/` 或为空）。
3. 点击 **上传文件**（或「上传」）。
4. 在弹窗里选择 **上传本地文件夹** 或 **逐层选择**：
   - 选中你本地的 **`out`** 文件夹**里面的全部内容**（`index.html`、`_next`、`detail` 等），
   - 不要只选 `out` 这个文件夹，而要进到 `out` 里面，全选后上传。
5. 上传方式选 **标准** 即可，点击 **上传**，等待完成。

### 方式 B：ossutil 命令行上传（适合经常更新）

1. **下载 ossutil**：  
   打开阿里云 OSS 文档，搜索「ossutil」，下载对应系统版本，或使用：  
   https://help.aliyun.com/document_detail/120075.html  

2. **配置 AccessKey**（在阿里云控制台 **AccessKey 管理** 里创建，并记住 AccessKey ID 和 Secret）：
   ```bash
   ossutil config
   ```
   按提示输入 Endpoint（如 `oss-cn-hangzhou.aliyuncs.com`）、AccessKey ID、AccessKey Secret。

3. **上传**（将 `你的Bucket名` 换成实际 Bucket 名称）：
   ```bash
   cd "/Users/bytedance/Desktop/My personal website/my-portfolio"
   ossutil cp -r out/ oss://你的Bucket名/ --update
   ```
   这会把 `out/` 下的内容同步到 Bucket 根目录。

---

## 第 5 步：用 OSS 默认地址访问（仅作验证）

1. 在 Bucket **概览** 或 **基础设置** 里找到 **Bucket 域名**（外网访问）：
   - 格式类似：`你的Bucket名.oss-cn-hangzhou.aliyuncs.com`
2. 用该域名访问时，**可能会触发浏览器下载 index.html 或白屏**，这是 OSS 默认域名的限制，不是配置错误。
3. 仅用于确认文件已上传成功（如用 curl 检查返回 200）。**对外分享请使用 GitHub Pages 或 Vercel 链接**，详见项目根目录 **`访问说明.md`**。
4. 若打不开（如 403），检查：Bucket 读写权限是否为 **公共读**；是否已设置默认首页为 `index.html`；根目录下是否有 `index.html`。

---

## 第 6 步：绑定自定义域名（可选，需已备案）

若你已有**已备案**的域名（如 `www.xxx.com`）：

1. 在 Bucket 管理页，左侧点击 **传输管理** → **域名管理**（或「绑定域名」）。
2. 点击 **绑定域名**，输入你的域名（如 `www.xxx.com`）。
3. 阿里云会给出该 Bucket 的 **CNAME 目标地址**（形如 `xxx.oss-cn-hangzhou.aliyuncs.com`）。
4. 到你的**域名解析**服务商（阿里云解析、腾讯云 DNSPod 等）：
   - 添加一条 **CNAME** 记录；
   - **主机记录**：若域名是 `www.xxx.com` 填 `www`，若是 `xxx.com` 填 `@`；
   - **记录值**：填阿里云给的 CNAME 目标地址。
5. 等待解析生效（几分钟到几小时），然后用 `http://你的域名` 访问。

若**暂不绑定域名**，请用 **GitHub Pages 或 Vercel** 链接分享网站（见 `访问说明.md`）；OSS 默认域名无法正常打开网页。备案并绑定域名后再做第 7、8 步即可用自定义域名 + CDN 访问。

---

## 第 7 步：开通并配置 CDN 加速

1. 阿里云控制台顶部搜索 **CDN**，进入 **CDN 产品控制台**（或 **全站加速** / **CDN**）。
2. 左侧点击 **域名管理** → **添加域名**。
3. 按下面填写：

   | 配置项 | 填写说明 |
   |--------|----------|
   | **加速域名** | 若在第 6 步绑了域名，填该域名（如 `www.xxx.com`）；若没绑，可先填一个你打算用来加速的域名（后面再去做解析）。 |
   | **业务类型** | 选 **图片小文件** 或 **静态页面**。 |
   | **源站信息** | 类型选 **OSS 域名**；在下来列表里选你刚创建的 Bucket（或手动填 Bucket 外网域名）。 |
   | **加速区域** | 选 **中国大陆**。 |
   | **资源组** | 默认即可。 |

4. 提交后，在 **域名管理** 列表里找到刚加的域名，点击 **管理**。
5. 在 **CNAME** 或 **解析设置** 里会看到 CDN 分配给你的 **CNAME 地址**（形如 `xxx.cdn.domain.com`）。
6. 到**域名解析**里：
   - 若加速域名是 `www.xxx.com`，添加 CNAME：主机记录 `www`，记录值填 CDN 的 CNAME 地址；
   - 若加速域名是 `cdn.xxx.com`，则主机记录填 `cdn`，记录值同样填 CDN 的 CNAME 地址。
7. 等待 CDN 状态变为 **正常**（约 5～10 分钟），然后用 **加速域名** 访问，如 `https://www.xxx.com`。

---

## 第 8 步：开启 HTTPS（推荐）

1. 在 CDN **域名管理** 里点击该域名的 **管理**。
2. 左侧找到 **HTTPS 配置**（或「证书管理」）。
3. 选择 **免费证书** 或 **上传证书**：
   - **免费证书**：阿里云可申请单域名免费 DV 证书，按提示申请并选择到该 CDN 域名。
4. 开启 **强制 HTTPS**（HTTP 自动跳 HTTPS）。
5. 保存后等证书生效，再用 `https://你的域名` 访问。

---

## 第 9 步：以后更新网站怎么操作

- **内容或代码更新**：本地重新执行 `npm run build`，然后按 **第 4 步** 再上传一次（覆盖 OSS 里原有文件）。
- 若用 **ossutil**：同样 `npm run build` 后执行：
  ```bash
  ossutil cp -r out/ oss://你的Bucket名/ --update
  ```
- CDN 若开启了**缓存**，更新后若看不到新内容，可在 CDN 控制台对该域名做 **刷新缓存**（URL 刷新或目录刷新）。

---

## 常见问题

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 打开 OSS 地址 403 | Bucket 不是公共读 | Bucket 读写权限改为「公共读」。 |
| 打开是下载或白屏 | OSS 默认域名会强制下载 HTML，属平台限制 | 分享请用 **GitHub Pages / Vercel** 链接（见 `访问说明.md`）；要正常打开 OSS 站需完成 **第 6 步绑定已备案自定义域名**，通过域名访问。 |
| 首页能开，点详情 404 | 未设置 404 页 | 默认 404 页设为 `index.html`。 |
| 绑了域名打不开 | 解析未生效或 CNAME 填错 | 检查解析记录、CNAME 是否指向 OSS/CDN 给的地址。 |
| 更新后还是旧内容 | CDN 缓存 | 在 CDN 控制台做 URL/目录刷新。 |

---

按以上步骤做完，方案二（阿里云 OSS + CDN）就部署完成，国内访问会走 CDN 节点，速度会比直连 GitHub Pages 更快。
