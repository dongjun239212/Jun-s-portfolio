# 国内 CDN/托管部署步骤（腾讯云 COS + CDN / 阿里云 OSS + CDN）

当流量变大或希望国内访问更稳定、更快时，可以把静态站部署到国内对象存储 + CDN。下面分别给出**腾讯云**和**阿里云**的详细步骤；二选一即可。

---

## 前置准备（通用）

1. **本地先构建出静态文件**
   ```bash
   cd "/Users/bytedance/Desktop/My personal website/my-portfolio"
   npm run build
   ```
   构建完成后，网站静态文件在 **`out/`** 目录下（含 `index.html`、`_next/`、`detail/`、图片等）。

2. **准备账号与域名（可选但推荐）**
   - 腾讯云：https://cloud.tencent.com 或 阿里云：https://www.aliyun.com
   - 若要用自己的域名（如 `www.xxx.com`），需先备案并拥有该域名的解析权限；未备案只能用平台提供的默认域名（如 `xxx.cos.ap-guangzhou.myqcloud.com` 或 `xxx.oss-cn-hangzhou.aliyuncs.com`）。

---

# 方案一：腾讯云 COS + CDN

## 一、开通 COS 并创建存储桶

1. 登录 [腾讯云控制台](https://console.cloud.tencent.com) → 搜索 **对象存储 COS** → 进入 **存储桶列表**。
2. 点击 **创建存储桶**：
   - **名称**：如 `my-portfolio`（全局唯一即可）。
   - **地域**：选离目标用户近的，如 **广州**。
   - **访问权限**：选 **公有读私有写**（网站要对外可读）。
   - **其他**：保持默认，创建。
3. 进入该存储桶 → **基础配置** → 找到 **静态网站**：
   - 开启 **静态网站**；
   - **索引文档**填：`index.html`；
   - **错误文档**（可选）填：`index.html`（便于 SPA/前端路由）；
   - 保存。

## 二、上传网站文件（out 目录）

1. 在存储桶内进入 **文件列表**，当前应在根目录。
2. 上传 **整个 `out` 目录里的内容**（不要上传 `out` 文件夹本身）：
   - 根目录要有：`index.html`、`_next/`、`detail/`、`favicon` 等；
   - 即：把本地 `out/` 下的所有文件和文件夹都选上并上传。
3. 上传方式任选其一：
   - **控制台**：在 COS 文件列表点 **上传** → 选本地 `out` 下全部内容；
   - **COSBrowser**：安装腾讯云 COSBrowser，选该桶，把 `out` 内所有内容拖进去。

## 三、绑定自定义域名（可选）

1. 在存储桶 **基础配置** → **域名与传输管理** → **自定义源站域名**：
   - 添加你的已备案域名（如 `www.xxx.com`）；
   - 按提示在域名服务商处添加 **CNAME** 记录，指向控制台给出的 COS 域名。
2. 若暂不绑定域名，可直接用 **默认访问域名**（格式：`<桶名>-<appid>.cos.<地域>.myqcloud.com`）访问，但需在「安全设置」中配置允许跨域等（见下）。

## 四、开启 CDN 加速（推荐）

1. 腾讯云控制台搜索 **CDN** → **域名管理** → **添加域名**：
   - **加速域名**：填你在 COS 里绑定的自定义域名，或先用 COS 默认域名（部分地域支持）；
   - **加速区域**：选 **中国境内**；
   - **源站类型**：选 **自有源站**；
   - **源站地址**：选对应 **COS 存储桶**（或填 COS 默认域名）；
   - **回源协议**：选 **HTTP** 即可。
2. 保存后，CDN 会分配一个 **CNAME**；到域名解析处把该加速域名 **CNAME** 到 CDN 给出的地址。
3. 等待 CDN 生效（约几分钟），之后国内访问会走 CDN 节点，速度更快。

## 五、访问网站

- **未绑域名**：浏览器打开 COS 提供的 **默认访问域名**（需确保桶为公有读、静态网站已开启）。
- **已绑域名 + CDN**：打开你配置的域名，如 `https://www.xxx.com`（若需 HTTPS，在 CDN 控制台申请免费证书并开启强制 HTTPS）。

# 方案二：阿里云 OSS + CDN

## 一、开通 OSS 并创建 Bucket

1. 登录 [阿里云控制台](https://ram.console.aliyun.com) → 搜索 **对象存储 OSS** → 进入 **Bucket 列表**。
2. 点击 **创建 Bucket**：
   - **Bucket 名称**：如 `my-portfolio`（全局唯一）；
   - **地域**：选 **华东1（杭州）** 等离用户近的；
   - **存储类型**：**标准存储**；
   - **读写权限**：**公共读**（网站需对外可读）；
   - 创建。

## 二、开启静态页面并设置默认首页

1. 进入该 Bucket → **基础设置** → **静态页面**：
   - **默认首页**：`index.html`；
   - **默认 404 页**（可选）：`index.html`（便于前端路由）；
   - 保存。

## 三、上传网站文件（out 目录）

1. 在 Bucket **文件管理** 中，当前应在根目录。
2. 上传 **`out` 目录下的全部内容**（不要带 `out` 这一层）：
   - 根目录包含：`index.html`、`_next/`、`detail/` 等；
   - 可用 **ossutil**、**控制台上传** 或 **ossbrowser** 批量上传。

**用 ossutil 示例（可选）：**
```bash
# 安装并配置 ossutil 后
ossutil cp -r out/ oss://你的Bucket名/ --update
```

## 四、绑定自定义域名（可选）

1. Bucket **传输管理** → **域名管理** → **绑定域名**：
   - 输入已备案域名；
   - 按提示在域名解析处添加 **CNAME** 到 OSS 提供的地址。
2. 若暂不绑定，可用 **Bucket 外网访问 Endpoint** 访问（格式：`https://<Bucket>.<Endpoint>`），需在 Bucket 读写权限为公共读且已设默认首页。

## 五、开启 CDN 加速（推荐）

1. 阿里云控制台 → **CDN** → **域名管理** → **添加域名**：
   - **加速域名**：填你在 OSS 绑定的域名（或支持用 OSS 外网 Endpoint 作为源时填该域名）；
   - **业务类型**：**图片小文件** 或 **静态页面**；
   - **源站信息**：类型选 **OSS 域名**，选择对应 Bucket；
   - **加速区域**：**中国大陆**。
2. 添加成功后，CDN 会分配 CNAME；到域名解析把加速域名 **CNAME** 到该地址。
3. 在 CDN 控制台可申请 **免费 HTTPS 证书** 并开启 **强制 HTTPS**。

## 六、访问网站

- **未绑域名**：用 OSS 提供的 **Bucket 外网访问地址** + 默认首页（如 `/index.html`）。
- **已绑域名 + CDN**：用你的域名访问，如 `https://www.xxx.com`。

# 与 GitHub Pages 的配合方式

- **日常开发**：继续在 GitHub 提交代码，Vercel / GitHub Pages 照常自动部署（海外 + 国内备用）。
- **国内加速**：在本地或 CI 中执行 `npm run build`，把生成的 **`out/` 内容** 同步到腾讯云 COS 或阿里云 OSS（可用脚本或 GitHub Actions 在 push 后自动上传），并配合 CDN。
- **双线**：海外用 Vercel 或 GitHub Pages 链接；国内把域名或单独域名解析到国内 CDN，实现「国内走 COS/OSS+CDN，海外走原部署」。

# 成本与注意点（简要）

| 项目       | 说明 |
|------------|------|
| **COS/OSS** | 按存储量、请求次数计费，个人静态站通常很低（几元/月级别）。 |
| **CDN**     | 按流量计费，有免费额度（如腾讯云/阿里云新用户），超出后按 GB 计费。 |
| **备案**    | 使用自有域名且解析到国内服务器/CDN，需完成 **ICP 备案**（在对应云厂商备案系统提交）。 |
| **HTTPS**   | 在 CDN 控制台申请免费证书并开启「强制 HTTPS」即可。 |

按上述步骤即可把当前静态站（`out/`）部署到腾讯云 COS + CDN 或阿里云 OSS + CDN，国内访问会更快、更稳定。
