## 阿里云 OSS 默认域名为何会下载 / 白屏？

使用 **OSS Bucket 自有域名**（如 `xxx.oss-cn-beijing.aliyuncs.com`）访问时，阿里云会强制在响应里加上「下载」相关头，导致：

- 浏览器把 `index.html` 当成**附件下载**，而不是渲染成页面  
- 即使不下载，也可能出现**白屏**（资源路径或策略导致）

这是**阿里云 OSS 的平台限制**，无法通过改上传脚本或对象元数据解决。

**正确用法**：若要用 OSS 做国内加速，需要先完成 **ICP 备案**，再在 OSS 控制台为 Bucket **绑定自定义域名**。通过自定义域名访问时，即可正常打开网页、不会下载。详见 `ALIYUN-OSS-CDN-步骤.md` 第 6、7、8 步。

## 一键部署到 Vercel（多一个可分享链接）

1. 打开 https://vercel.com 并登录（可用 GitHub 账号）。
2. 点击 **Add New → Project**，选择你的 GitHub 仓库 `My-new-portfolio`。
3. 构建配置保持默认即可（Vercel 会识别 Next.js）：
   - Build Command: `npm run build`
   - Output Directory: `out`
4. 点击 **Deploy**，等待完成后会得到 `https://你的项目名.vercel.app`。
5. 之后每次推送到 GitHub，Vercel 会自动重新部署。

该链接可正常打开网页，适合作为备用或海外访问。

## 为什么关掉 VPN 后加载变慢？

**原因**：GitHub Pages、Vercel 等服务的服务器在海外。国内访问时，请求要跨国际网络，延迟高、带宽受限，所以关掉 VPN 直连会明显变慢。这是**托管位置**导致的，单靠前端代码优化无法根本解决。

**代码层面已做的优化**（能减轻体感，但无法消除网络延迟）：

- 字体 `display: swap`，避免等字体导致白屏
- 首屏 Hero 图高优先级、首屏外图片懒加载
- 已去掉 Google 等境外依赖，图片已压缩

要进一步提速，需要让**网站从国内节点分发**。

## 加载速度优化与国内加速方案

### 方案一：国内加速（效果最好，需备案）

完成 **ICP 备案** 后，按 `ALIYUN-OSS-CDN-步骤.md`：

1. 为 OSS Bucket **绑定已备案的自定义域名**
2. 在 CDN 里添加该域名，源站选你的 OSS
3. 用 **CDN 域名** 访问站点

这样流量走国内 CDN 节点，关掉 VPN 后速度会明显提升。

### 方案二：Gitee Pages（国内托管，无需备案）

[Gitee（码云）](https://gitee.com) 的 Pages 服务在国内，用其默认域名 `xxx.gitee.io` 访问时，国内直连通常比 GitHub Pages 快。

1. 注册 Gitee，新建仓库（可私有）。
2. 本地构建：`npm run build`（不要加 `GITHUB_PAGES=true`，即不加 basePath）。
3. 把 **`out` 目录里的全部内容** 推送到仓库根目录（或建一个分支专门放静态文件）。
4. 在仓库 **服务 → Gitee Pages** 里选择分支/目录，开启 Pages。
5. 用生成的 `https://你的用户名.gitee.io/仓库名/` 访问（若仓库名与用户名一致，则为 `https://用户名.gitee.io/`）。

注意：Gitee 免费版 Pages 有时需手动「更新」才会发布最新内容；若用自定义域名，可能需备案。

### 方案三：继续用 GitHub / Vercel

- 适合不介意国内稍慢、或主要给海外/挂 VPN 的用户访问。
- 分享时仍用：https://dongjun239212.github.io/My-new-portfolio/ 或 Vercel 链接。
